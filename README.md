# Lusha ToGo

A React Native mobile app for the [Lusha](https://www.lusha.com) B2B sales intelligence platform — bringing premium prospect search, company discovery, and lead management to Android.

---

## Features

- **AI-Powered Search** — describe who you're looking for in plain text; structured filters are extracted automatically
- **Contact Search** — filter by job title, seniority, department, location, and more
- **Company Search** — filter by industry, size, revenue, location, and founded year
- **Recommended Leads** — personalized ICP-based recommendations on the home screen
- **Lists Management** — view and manage your Lusha prospect lists
- **Contact & Company Detail Pages** — full profile view with reveal/unmask support
- **OAuth Login** — WebView-based auth with cookie session persistence

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | React Native + Expo (SDK 52) |
| Navigation | Expo Router (file-based) |
| Styling | NativeWind (Tailwind for RN) |
| State | Zustand |
| Server State | TanStack Query (React Query v5) |
| HTTP | Axios |
| Auth | WebView OAuth + `@react-native-cookies/cookies` |
| Storage | Expo SecureStore |
| Build | Gradle (debug APK via Metro bundler) |

## Getting Started

### Prerequisites

- Node.js 18+
- Expo CLI: `npm install -g expo-cli`
- Android Studio + Android SDK (for emulator)
- ADB (Android Debug Bridge) for physical device installs

### Install dependencies

```bash
npm install
```

### Run on emulator (Metro dev server)

```bash
npx expo start --android
```

### Build a debug APK and install on device

```bash
bash build_and_deploy.sh
```

This script:
1. Auto-bumps the patch version in `app.json` and `package.json`
2. Runs `expo prebuild` to generate the native Android project
3. Builds a debug APK via Gradle (no ProGuard/R8 — fast builds)
4. Installs via ADB on the connected device (prefers physical device over emulator)

> **Note:** The `android/` directory is gitignored (regenerated by `expo prebuild`). Run `expo prebuild --platform android` after cloning before building natively.

## Pre-built APK

A ready-to-install debug APK is included at the repo root:

```
lusha-togo.apk
```

Install directly via ADB:

```bash
adb install -r lusha-togo.apk
```

Or transfer to your Android device and install manually (enable "Install from unknown sources" in Settings).

> This is a **debug build** connected to the Lusha production API. You will need a valid Lusha account to log in.

## Authentication

The app uses WebView-based OAuth. On first launch, a login screen is presented (`auth.lusha.com`). After successful login, session cookies are captured and stored securely via `expo-secure-store` for subsequent app launches.

---

## API

All API calls go through `src/api/client.ts` (Axios instance pointed at `https://dashboard-services.lusha.com`). Authentication is cookie-based — the WebView login flow captures the session cookies which are then injected into all subsequent Axios requests.
